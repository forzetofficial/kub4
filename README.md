# Исходные данные

# Компонент	ОЗУ	CPU	Кол-во копий	Отказоустойчивость

# База данных	4 ГБ	1 ядро	3	Да

# Кеш	4 ГБ	1 ядро	3	Да

# Фронтенд	50 МБ	0.2 ядра	5	Нет

# Бекенд	600 МБ	1 ядро	10	Нет

# 1\. Расчёт суммарных ресурсов

# ОЗУ

# БД: 4 ГБ × 3 = 12 ГБ

# 

# Кеш: 4 ГБ × 3 = 12 ГБ

# 

# Фронтенд: 0.05 ГБ × 5 = 0.25 ГБ

# 

# Бекенд: 0.6 ГБ × 10 = 6 ГБ

# 

# Итого: 30.25 ГБ

# 

# CPU

# БД: 1 × 3 = 3 ядра

# 

# Кеш: 1 × 3 = 3 ядра

# 

# Фронтенд: 0.2 × 5 = 1 ядро

# 

# Бекенд: 1 × 10 = 10 ядер

# 

# Итого: 17 ядер

# 

# 2\. Учёт overhead (15%)

# ОЗУ: 30.25 × 1.15 = 34.79 ГБ ≈ 35 ГБ

# 

# CPU: 17 × 1.15 = 19.55 ядер ≈ 20 ядер

# 

# 3\. Системные компоненты

# Компонент	ОЗУ	CPU

# kubelet, container runtime	1 ГБ	0.5 ядра

# Monitoring (Prometheus/Grafana)	2 ГБ	1 ядро

# Логирование	2 ГБ	1 ядро

# Ingress Controller	1 ГБ	0.5 ядра

# Итого	6.5 ГБ	3.5 ядра

# 4\. Итого ресурсов кластера

# Категория	ОЗУ	CPU

# Приложения	35 ГБ	20 ядер

# Системные	6.5 ГБ	3.5 ядра

# Итого	41.5 ГБ	23.5 ядра

# Учёт отказа ноды (N+1, +20%)

# ОЗУ: 41.5 × 1.2 = 49.8 ≈ 50 ГБ

# 

# CPU: 23.5 × 1.2 = 28.2 ≈ 29 ядер

# 

# 5\. Выбор нод

# Вариант: Одинаковые ноды (рекомендуется)

# Характеристики ноды: 8 vCPU, 32 ГБ ОЗУ, 100 ГБ SSD

# 

# Расчёт:

# 

# По ОЗУ: 50 / 32 = 1.56 → 2 ноды (минимум)

# 

# По CPU: 29 / 8 = 3.63 → 4 ноды

# 

# Итого: 4 ноды по 8 vCPU, 32 ГБ ОЗУ

#
